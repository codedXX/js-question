# 用“排队”通俗理解 Vue3 快速 Diff 算法

太好了，既然你是新手，我们就完全抛开那些复杂的专业术语，用**“排队”**的通俗比喻来聊聊 Vue3 的 Diff 算法。

### 什么是 Diff 算法？它在干嘛？
想象一下，由于某个原因，你需要重新安排一队排好的小朋友（这队小朋友就是**网页上的旧节点**）。
你会得到一张“新队伍的名单”（也就是**新节点**）。

最笨的办法是：让旧队伍的小朋友全解散（销毁旧 DOM），然后按照新名单重新叫小朋友一个个排好（新建新 DOM）。但在网页里，**新建和销毁节点非常耗时（性能差）**。

聪明的办法就是 Diff 算法的终极目标：**找不同，尽量让小朋友呆在原地，只移动位置变化了的小朋友，或者只加新来的、赶走不需要的。**

Vue3 的 Diff 算法（官方叫快速 Diff，Fast Diff）在找不同时，梳理了所有可能的情况，分成了**三步走**：

---

### 第一步：试图“掐头去尾”（走捷径优化）
如果队伍中间乱了，但两头可能没变，Vue3 会试着先处理两头，这是为了提升更新速度的捷径。

**可能性 1：两端没有发生变化（捷径走通）**
*   **旧队伍：** **A B** C D E **F G**
*   **新队伍：** **A B** X Y Z **F G**
*   **处理机制：** Vue3 从左往右看，发现 A 和 B 没变，定在原地；从右往左看，发现 F 和 G 没变，也定在原地。问题立刻简化为只处理中间的 `C D E` 和 `X Y Z`。这在实际网页里非常常见（比如在列表末尾追加元素）。

**可能性 2：两端发生了变化（捷径中断）**
*   **旧队伍：** **A** B C **D**
*   **新队伍：** **X** B C **Y**
*   **处理机制：** Vue 一看开头，旧 A 和新 X 不一样，立刻停止头部对比；一看结尾，旧 D 和新 Y 不一样，立刻停止尾部对比。
*   **结果：** 掐头去尾失败。没关系，这并不影响程序的运行，只是意味着这个队伍没有捷径可走，**所有的节点都会被当成“一团乱麻”直接留给第三步处理**。

---

### 第二步：多退少补（处理仅剩一头的情况）
掐头去尾之后，如果发现某一边被掏空了（只剩下纯新增或纯删除的情况），处理起来就很简单粗暴。

**可能性 1：纯新增节点**
*   **旧队伍：** A B （中间空了）
*   **新队伍：** A B **C D**
*   **处理机制：** 旧节点处理完了，新节点还有剩。这时候 Vue 直接把剩下的 C 和 D 作为一个个新节点插入到页面里。

**可能性 2：纯删除节点**
*   **旧队伍：** A B **C D**
*   **新队伍：** A B （中间空了）
*   **处理机制：** 新节点梳理完了，旧节点还有剩。这时候 Vue 直接把多余的 C 和 D 从网页上彻底删除。

---

### 第三步：处理“一团乱麻”（终极大招：最长递增子序列）
如果前两步走完，发现旧队伍和新队伍都有剩余，情况错综复杂（有人新来、有人走开、有人换位置），这就轮到终极武器出场了。

> **附：什么是最长递增子序列（LIS）？**
> 这个概念非常直观：给你一个数组，比如 `[10, 9, 2, 5, 3, 7, 101, 18]`。你要在里面挑出一些数字，保持它们原来的先后顺序，并且这些数字必须是**越来越大**的。在这个例子里，最长的递增子序列是 `[2, 3, 7, 101]` 或者 `[2, 5, 7, 101]`（甚至是 `[2, 3, 7, 18]`），长度都是 4。所以，理解 LIS 是什么一点都不难。它的核心理念就是**找到一截相对顺序没有被打乱的最长队伍**。

**综合大例子（涵盖两端改变、中间乱序、增删节点）：**
*   **中间剩下的旧队伍：** A B C D E
*   **中间剩下的新队伍：** X C B D Y

这就全乱套了！两端变了（A变成了X，E变成了Y），中间顺序变了（B和C互换），有人走（A,E），有人来（X,Y）。Vue3 是这么做的：

1.  **先做个记录单：** 看看新队伍里的人在旧队伍里站在第几个位置。
    在这份新名单 `[X, C, B, D, Y]` 里：
    *   X：新来的，查不到位置
    *   C：原来在第 3 局
    *   B：原来在第 2 局
    *   D：原来在第 4 局
    *   Y：新来的，查不到位置
    *   *至于旧队伍的 A 和 E，因为在新名单里没找到，直接被开除（删除）！*
    此时能查到的旧位置顺序是：`[3, 2, 4]` (对应新名单里的 C, B, D)。

2.  **找出“最懒的一拨人”（LIS）：**
    在 `[3, 2, 4]` 这个顺序里，Vue 要找出一个**递增子序列**。
    在这里，可以是 `[2, 4]`（对应 B 和 D），也可以是 `[3, 4]`（对应 C 和 D）。这意味着，如果只看 B 和 D，它俩的相对前后顺序没变。
    所以，**B 和 D（或者C和D） 就是那个相对位置没变的一组人**。

3.  **让“懒人”不动，其他瞎跑的人去挪位置：**
    Vue 会下达指令：
    *   新来的 X：你自己去队伍最前面插好！
    *   新来的 Y：你自己去队伍最后面插好！
    *   老节点 B 和 D：“你们俩相对位置没变，钉在原地别动！”
    *   老节点 C：“你位置乱了，跑去 B 的前面待着！”

### 总结一下
Vue3 Diff 算法的通俗流程就是：
1.  **试图掐头去尾**（走捷径找出没变的，如果全变了就直接跳过）。
2.  **多退少补**（如果剩下的全是新增或删除，直接批量干活）。
3.  **对付一团乱麻**（利用最长递增子序列，在乱局中找到一帮**连相对顺序都没变的人**把他们固定死，只要求剩下乱跑的人挪动）。

正是有了终极大招“最长递增子序列”的神操作，Vue3 成功地应对了无论多复杂的更新局面，做到了：**能不挪动的节点坚决不挪动**，从而极大提升了页面的更新速度！
